git笔记

2013-06-25

一些全局配置

git config --global push.default matching

git config --global color.status auto
git config --global color.diff auto
git config --global color.branch auto
git config --global color.interactive auto

添加所有新文件

git add .

从版本中去除一个文件

git rm test.html.rb

假设有很多新文件和删除了好多文件

git add -A

获取所有远程版本

git fetch

创建并切换到新分支

git checkout -b newbranch

切换到分支

git checkout master

迁移原有git项目到新得项目地址

git remote set-url origin git@git.github.com:welsonla/studio.git

查看当前git分支

git branch -r

查看git log

git log

查看当前tag

git tag

创建一个新的tag

git tag v1.1

降当前tag推送到服务器

git push origin --tags

推送单个
git push origin v1.1

将现有svn项目转换成git项目，并保留所有commit log

git svn clone [SVN-Path]

回滚

#http://stackoverflow.com/questions/1338728/delete-commits-from-a-branch-in-git
git reset --hard commitid
git push --force

ignore

.gitignore 新文件使用正则匹配
*.file_extension
.idea/*
remove exist files(现有文件需要软删除)
git rm -r --cached . //soft delete
git add .
git commit -m 'remove ignore files'
